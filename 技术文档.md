## 3. 数据库设计

### SQLite 表结构

```sql
-- 面试题表
CREATE TABLE questions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    category VARCHAR(50) NOT NULL,      -- Agent/RAG/Memory/Tool Calling
    question TEXT NOT NULL,
    answer TEXT,
    source VARCHAR(20) DEFAULT 'system', -- system: 系统题库, generated: LLM生成
    resume_id VARCHAR(50),              -- 简历关联ID (简历面试时使用)
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 收藏表 (可选，也可直接在 questions 表增加 is_favorited 字段)
CREATE TABLE favorites (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    question_id INTEGER NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (question_id) REFERENCES questions(id)
);
```

### ChromaDB Collection

```python
# Collection: questions
{
    "id": "1",                    # 对应 SQLite question.id
    "embedding": [0.1, 0.2, ...], # Question + Answer 的 embedding
    "metadata": {
        "category": "Agent",
        "source": "system",
        "question": "...",
        "answer": "..."
    }
}
```

## 4. API 设计

### 题库管理

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/questions?category=Agent&limit=5` | 获取指定分类的5个问题 |
| POST | `/api/questions/generate` | LLM 生成新问题 |
| POST | `/api/questions/{id}/favorite` | 收藏问题 |
| DELETE | `/api/questions/{id}` | 删除问题 |
| PUT | `/api/questions/{id}/regenerate` | 重新生成答案 |

### 简历面试

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/resume/generate` | 根据项目描述生成问题 |
| POST | `/api/resume/{id}/followup` | 生成追问问题 |

## 5. 核心业务流程

### 题库管理流程

```
用户选择分类
     ↓
点击"下一批" / "新问题"
     ↓
┌─────────────────────────────────────────┐
│ 下一批: SQLite 查询 → 返回问题列表        │
│ 新问题: LLM 生成 → 返回新问题              │
└─────────────────────────────────────────┘
     ↓
用户点击"收藏"
     ↓
SQLite INSERT + ChromaDB ADD (同步)
     ↓
显示已收藏状态 (按钮变为: 生成新答案/删除)
```

### 简历面试流程

```
用户输入项目描述
     ↓
点击"生成面试题"
     ↓
LLM 提取关键点 → 生成 3 个问题及答案
     ↓
返回问题列表 (收藏/删除按钮)
     ↓
点击"下一批"
     ↓
LLM 深挖/扩展 → 再次生成 3 个问题
```
